## 5. Insights and Learnings

- **Technical:** The `gh project item-list` CLI command silently truncates at page 1 (30 items). For any board with more than 30 items, GraphQL is the only reliable query method. This is a significant footgun for automation.
- **Process:** The work-ticket → review-pr → human-merge → agent-bookkeeping cycle is smooth when the human confirms merges promptly. The post-merge handoff (Memory MCP entity + board move to Done) should be automatic on "I merged it" — no prompting.
- **Framework:** The naming convention table with `{domain}` prefix is a force multiplier for keyword search. Without it, agents would name entities inconsistently and retrieval quality would degrade over time.
- **Process:** The PR reviewer agent's suggestion about updating the Product Manager agent file for `FeatureDecision` naming is a valid follow-up — not addressed in this session.
