# Distribution: Framework / User-Content Boundary

This document classifies every file and directory in the Agile Flow template
repository. The classification drives update behavior: which files are safe
to overwrite during framework updates, which must never be touched, and
which contain both framework and user sections.

## Categories

| Category | Meaning | Update Behavior |
|----------|---------|-----------------|
| **framework** | Controlled by Agile Flow | Safe to overwrite during sync |
| **user-content** | Owned by the user | Never modified by updates |
| **hybrid** | Both framework and user sections | Only framework-marked sections updated |

## Marker Convention

Hybrid files use markers to delineate framework-controlled sections.

| File Type | Start Marker | End Marker |
|-----------|-------------|------------|
| Markdown (`.md`) | `<!-- FRAMEWORK:START -->` | `<!-- FRAMEWORK:END -->` |
| Shell / YAML | `# FRAMEWORK:START` | `# FRAMEWORK:END` |
| TypeScript / JS | `// FRAMEWORK:START` | `// FRAMEWORK:END` |
| JSON | Uses template/local split pattern | (no inline markers) |

Content **outside** markers in hybrid files is user-owned. Content
**inside** markers may be updated by the template-sync mechanism.

## Sync Directories Summary

These directories contain only framework files and are safe for bulk sync:

```json
[
  ".claude/agents",
  ".claude/commands",
  ".claude/hooks",
  ".claude/skills",
  "scripts",
  "starters",
  "docs/TICKET-FORMAT.md",
  "docs/AGENTIC-CONTROLS.md",
  "docs/CONTEXT-OPTIMIZATIONS.md",
  "docs/AGENT-WORKFLOW-SUMMARY.md",
  "docs/ARTIFACT-FLOW.md"
]
```

Hybrid files require marker-aware merging and are handled separately by
the template-sync mechanism.
