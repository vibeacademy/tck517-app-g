## Market Research

Conduct structured market research to produce an evidence-based landscape analysis. Output feeds into `/bootstrap-product` to replace guesswork with data.

## Instructions for the Agent

**IMPORTANT: Follow this exact sequence. Ask questions ONE AT A TIME and wait for responses before proceeding.**

For multiple-choice questions, present numbered options and ask the user to respond with the number OR type their own answer.

---

## Questionnaire

**Question 1**
```
What product or idea are you researching?

Describe it in one sentence.

Example: "A fitness studio management platform for boutique gym owners"
Example: "A CLI tool that automates database migration testing"
```

**Question 2**
```
What industry or market does this product belong to?

1. SaaS / Cloud Software
2. Developer Tools
3. E-commerce / Marketplace
4. Healthcare / Wellness
5. Education / EdTech
6. Finance / Fintech
7. Consumer Mobile
8. Other (please describe)

Enter a number (1-8) or describe:
```

**Question 3**
```
List any competitors you already know about (even vaguely).

If none, write "None" — the research phase will find them.
```

**Question 4**
```
Who do you think the target audience is?

Example: "Small business owners with 1-10 employees"
Example: "Backend developers at mid-size companies"

If unsure, write "Not sure yet"
```

**Question 5**
```
What geographic focus, if any?

1. Global / no specific focus
2. United States
3. Europe
4. Asia-Pacific
5. Specific country or region (please specify)

Enter a number (1-5) or describe:
```

**Question 6 (Optional)**
```
Do you have any specific URLs you'd like included in the research?
(articles, competitor pages, market reports, pricing pages, etc.)

Paste up to 5 URLs, one per line.

Type "skip" to continue without.
```

## After Collecting All Responses

### Phase 1: Web Research

Attempt to use `WebSearch` to gather real market data. Run searches in parallel where possible:

**Market overview searches (parallel):**
- `"{industry} market size {current year}"` — market size and growth
- `"{industry} trends {current year}"` — current trends and shifts
- `"{product type} market landscape"` — landscape overview

**Competitor searches (parallel):**
- `"{competitor name} review"` for each named competitor
- `"best {product type} tools {current year}"` — discover competitors the user missed
- `"alternatives to {known competitor}"` — find additional players

**Audience searches (parallel):**
- `"{target audience} pain points {industry}"` — audience challenges
- `"{target audience} software adoption"` — buying behavior

For each search result, note the source URL for the Sources section.

**User-provided URLs (from Q6):**

If the user provided URLs in Q6, fetch each one using `WebFetch` after the `WebSearch` queries complete. Run fetches in parallel where possible:

- Extract key findings from each URL (market data, competitor info, pricing, audience insights)
- Categorize findings by the section they inform (Market Overview, Competitor Analysis, Target Audience)
- Use the original URL as the source attribution in the Sources section

**If `WebSearch` is unavailable:** Skip web search entirely. Instead, conduct a structured interview — ask 3-5 follow-up questions to extract what the user already knows about the market, competitors, and audience. Note all findings as "User-reported (not web-validated)" in the confidence labels. However, if the user provided URLs in Q6, still fetch those using `WebFetch` — URL fetching does not depend on `WebSearch` availability.

### Phase 2: Synthesize

Combine user answers, web research findings, and any content fetched from user-provided URLs. For each section, assign a confidence label:

| Label | Meaning |
|-------|---------|
| Research-backed | Supported by web search results with sources |
| Partially validated | Some supporting evidence found, gaps remain |
| Needs validation | Based on user input only, no web confirmation |

**IMPORTANT: Never fabricate data.** If a search returns no results for a topic, state "No data found" rather than inventing statistics or facts.

### Phase 3: Write Output

Present a summary to the user and confirm before writing the file.

Write the output to **`docs/MARKET-RESEARCH.md`** using the template below.

## What Happens Next

After completing market research:
1. Run `/jtbd` to analyze user jobs and pain points (uses this research as input)
2. Run `/positioning` to define product positioning (uses both prior artifacts)
3. Run `/bootstrap-product` to create the PRD (reads all three artifacts and pre-populates questions)
